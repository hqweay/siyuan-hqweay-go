### i18n enforcement

All user-visible strings in plugin `.tsx`/`.ts`/`.svelte` files MUST go through `this.t("key")`.
Hardcoded Chinese characters (or any non-ASCII user-facing text) in UI or logic files are forbidden.
Translations belong in `src/translations/parts/[plugin-name].ts` with keys prefixed by plugin name.
Before finalising any sub-plugin, run:
```bash
grep -rn '[^\x00-\x7F]' src/lets-[plugin]/ --include="*.tsx" --include="*.ts" --include="*.svelte"
```
This must return zero matches in all files except the translation file itself.

### Sub-Plugin Development Guidelines

1. **Sub-Plugin Inheritance & Registration**:
   - All sub-plugins (located in `src/lets-*`) must extend `SubPluginBase` from `src/libs/sub-plugin-base.ts`.
   - Never manually instantiate settings or register event listeners that leak memory. `onload()` and `onunload()` stubs are provided by the base class.
   - Sub-plugins must be registered via `PluginRegistry` to receive injected runtime helpers (`this.t`, `this.getSetting`, `this.setSetting`). Never attempt to call these helpers outside the registry context.

2. **Modular Type-Safe i18n Configuration**:
   - **File Location**: Put all translations for sub-plugin `lets-[name]` in `src/translations/parts/lets-[name].ts`.
   - **Strict Type Parity**: The file must export two objects: `en` and `zhCN: typeof en` to enforce key symmetry at compile time.
   - **Do NOT manually edit** `src/translations/en.ts` or `src/translations/zh-CN.ts`. These entrypoints are automatically generated by the Vite `i18n-generate` compiler plugin at build/dev start.
   - **Auto-completion**: In code, use the type-safe `TranslationKey` union (injected automatically in `SubPluginBase`) to get auto-completion and key check safety.
   - **Git Hygiene**: Generated files (`src/translations/en.ts`, `zh-CN.ts` and `public/i18n/*.json`) are blacklisted in `.gitignore`. Do not track them in Git.
   - **Settings Dynamic Translation**: 
     - Inside `plugin.ts` settings metadata, use translation keys (e.g. `lets-voiceNotes.token`) for option `title`, `description`, and `value` (for button types). They are automatically resolved by `setting-item.svelte` at render time.
     - **CRITICAL**: Do NOT translate group names (e.g., `"开关"`, `"设置"`) and sub-plugin `displayName` strings inside raw data objects, as settings click event comparisons (e.g., `if ("VoiceNotes 同步" === detail.group)`) rely on exact untranslated keys. Translate them only visually in Svelte markup using the `getGroupLabel` helper in `setting.svelte`.

3. **Release & Update Alerts Workflow**:
   - Always bump versions using Changesets (`npx changeset` / `npx changeset version`).
   - Run `pnpm release` (which wraps `update-version.sh`) for interactive bumps and release commits.
   - The `update-changelog-data.cjs` script extracts changes for the GitHub Release Action.
   - The `lets-alert` sub-plugin parses `CHANGELOG.md` dynamically at runtime, displaying all release notes merged from `currentVersion` down to the user's `lastVersion`.
